# cursor.directory Guide

> **TL;DR**: How to use cursor.directory — the community hub with 75K+ developers, 100+ plugins, 75+ MCP servers.

## What Is cursor.directory?

[cursor.directory](https://cursor.directory) is the central community hub for Cursor AI:
- **Plugins**: 100+ community and official plugins
- **MCP Servers**: 75+ external tool integrations
- **Rules**: .mdc files for various frameworks
- **Events**: Community meetups
- **Jobs**: AI engineering positions

## How to Browse

### Plugins
1. Go to cursor.directory
2. Browse plugins by category
3. Click for details: description, install command, what's included
4. Install: `/add-plugin [name]` in Cursor

### MCP Servers
1. Filter by "MCP Servers"
2. View tools count, transport type
3. One-click install or copy config to `.cursor/mcp.json`

### Rules
1. Filter by "Rules"
2. Browse by framework/language
3. Preview .mdc content
4. Download and copy to `.cursor/rules/`

## How to Submit

1. Go to [cursor.directory/plugins/new](https://cursor.directory/plugins/new)
2. Sign in with GitHub or Google
3. Your repo is scanned for Open Plugins standard components:
   - `rules/*.mdc` → auto-detected as rules
   - `SKILL.md` → auto-detected as skill
   - `mcp.json` → auto-detected as MCP config
4. Review and publish

## Tips

- **Star count matters**: Higher-starred plugins tend to be better maintained
- **Check update date**: Prefer recently updated plugins
- **Read the source**: Plugins are open source — review before installing
- **Community-driven**: Contributions welcome, submit your own rules and plugins
